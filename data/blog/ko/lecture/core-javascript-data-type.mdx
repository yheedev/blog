# 1. 데이터 타입의 종류

기본형(= primitive, 원시형)은 할당이나 연산시 복제된다. 참조형(= reference)은 참조된다.

둘 다 복제를 하긴 하지만, 기본형은 값이 담긴 주솟값을 바로 복제한다. 반면에 참조형은 값이 담긴 주솟값들로 이루어진 묶음을 가리키는 주솟값 자체를 복제한다.

기본형은 불변성(immutability)을 띈다. 불변성을 이해하기 위해서는 메모리와 데이터, 식별자와 변수 개념을 구분할 수 있어야 한다.

# 2. 데이터 타입에 관한 배경지식

- 비트: 0 또는 1만 표현할 수 있는 하나의 메모리 조각.
- 메모리: 매우 많은 비트들으로 구성되어있다.

각 비트는 고유한 식별자를 통해 위치를 확인할 수 있다. 비트 하나 하나를 통해 위치를 확인하는 것은 비효율적일 수 있다. 만약 묶어서 하나의 단위를 사용한다면?

장점: 표현할 수 있는 값도 늘어나고, 검색 시간도 줄일 수 있다.
단점: 동시에 낭비되는 비트가 생기기도 한다. 자주 사용하지 않을 데이터를 표현하기 위해 빈 공간을 방치학 보다는 표현 가능한 개수에 어느 정도 제약이 따르더라도 크게 문제 되지 않을 적정 공간을 묶는 편이 낫다.

위와 같은 고민의 결과로 바이트(byte) 단위가 생겼다. 1바이트는 8비트이다. 그래서 1바이트는 총 256(2^8)개의 값 표현 가능. 2바이트는 65536(2^16)개의 값 표현 가능.

<hr className="invisible mt-1" />

C, C++, Java 같은 정적 타입 언어들은 메모리 낭비를 최소화하기 위해 데이터 타입별로 할당할 메모리 영역을 2바이트, 4바이트 등으로 나눠서 정해두었다.
그래서 2바이트 크기의 정수형 타입(short)는 -32,768 ~ 32,767 사이의 값만 표현할 수 있다. 만약 이 범위를 벗어나는 값을 저장하려고 하면 오버플로우(overflow)가 발생한다.
이 문제를 해결하기 위해서는 사용자가 직접 4바이트 크기의 정수형 타입(int) 등으로 형변환을 해야 했다. UX 측면에서는 불편했지만, 메모리 용량이 부족했던 시절에는 불가피한 선택이었다.

<hr className="invisible mt-1" />

현재 메모리 용량에 대한 압박이 과거에 비해 줄어든 상황에서 자바스크립트는 상대적으로 메모리 관리에 대한 압박에서 자유로운 편이다. 그래서 메모리 공간을 좀 더 넉넉하게 할당한다. 숫자의 경우 64비트, 즉 8비트를 확보한다.
덕분에 형변환 없이도 매우 큰 숫자부터 매우 작은 숫자까지 표현할 수 있게 되었다. (약 -9,007,199,254,740,992 ~ 9,007,199,254,740,992)

<hr className="invisible mt-1" />

비트는 고유한 식별자를 지닌다. 바이트 역시 시작하는 비트의 식별자로 위치를 파악할 수 있다. 모든 데이터는 바이트 단위의 식별자, 더 정확히 **메모리 주솟값(memory addres)**을 통해 서로 구분하고 연결할 수 있다.

<hr className="invisible mt-1" />

- 변수 (variable): 변할 수 있는 데이터(문자열, 객체, 배열 등)
- 식별자 (identifier): 어떤 데이터를 식별하는 데에 사용하는 이름. 즉 변수 이름.
